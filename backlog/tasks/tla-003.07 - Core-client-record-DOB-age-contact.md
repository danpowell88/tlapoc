## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As a** front-desk/clinician, **I want** a single client record with identity, contact and date of birth, **so that** every booking, intake, treatment and payment attaches to the right person.

The core Client entity the whole platform hangs off: name, contact details, date of birth with a derived under-18 flag (feeds cooling-off and consent rules), and basic flags. The richer client 360 view lives in the booking epic; this is the underlying record and its creation.

**Surface:** web (staff) · API · **Touches:** client record, DOB/age derivation.

**Build steps (functional):**
- [ ] Create and edit a client with identity and contact details
- [ ] Capture date of birth and derive an under-18 flag
- [ ] The record is tenant-scoped and audited on read/write
- [ ] Other features reference this single client record

**References:** PRD-01 §6 (client DOB/age), REQ-CLI-3; under-18 feeds C6/C9.
