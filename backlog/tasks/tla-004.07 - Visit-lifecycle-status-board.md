## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As a** front desk, **I want** each appointment to move through clear statuses with a today board, **so that** the whole team knows who is booked, waiting, in-room and done.

The visit state-machine and the board that runs the day: booked -> checked-in -> in-room -> complete, plus late and no-show. Check-in on arrival; an 'in-room now' indicator with quick links to chart/profile; row actions per status; VIP/first-time tags. A no-show raises a follow-up call task.

**Surface:** web (staff) · API · **Touches:** appointment status state-machine, today board, follow-up task.

**Build steps (functional):**
- [ ] Status state-machine: booked, checked-in, in-room, complete, late, no-show
- [ ] Check a client in on arrival
- [ ] Today board shows each visit's status with role-appropriate row actions
- [ ] 'In-room now' indicator links to charting/profile
- [ ] A no-show raises a follow-up task; VIP/first-time tags display

**Prototype:** schedule.png / checkin.png — status flow.
**References:** PRD-02 §6 (lifecycle), REQ-BOOK-4/6/7, ADR-0024.
