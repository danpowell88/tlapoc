## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As a** prescriber, **I want** to record a synchronous consult before any script, **so that** every prescription is provably tied to a real assessment of this patient.

The Consult entity: modality (in-person or recorded-from-external-telehealth), prescriber identity, timestamp, external reference and notes. A consult is required before a prescription can exist, and it is what satisfies the booking/charting consult gate. Single-product telehealth is treated as a configurable regulatory risk.

**Surface:** web (staff/prescriber) · API · **Touches:** consult record, links to appointment/client.

**Build steps (functional):**
- [ ] Record a consult with modality, prescriber, timestamp, external ref and notes
- [ ] A consult links to the client and the visit and satisfies the consult gate
- [ ] Telehealth-external path captures the external reference
- [ ] Remote/telehealth path is configurable as a regulatory-risk toggle

**References:** PRD-04 §6 (consult), REQ-RX-1/4, C1, AC1.
<!-- SECTION:DESCRIPTION:END -->
