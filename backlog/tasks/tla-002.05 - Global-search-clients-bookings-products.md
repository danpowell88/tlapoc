## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As a** staff member, **I want** one search box to find a client, booking or product fast, **so that** I am not clicking through menus while someone waits at the desk.

A global search that returns clients, appointments and catalogue items, respecting tenant scope and role capabilities (a result a role cannot view is not returned). Optimised for the front-desk 'someone is on the phone' moment.

**Surface:** web (staff) · API · **Touches:** search index across key entities, capability filtering.

**Build steps (functional):**
- [ ] One search entry returns clients, bookings and products
- [ ] Results are scoped to the tenant and the role's capabilities
- [ ] Selecting a result navigates to that record
- [ ] Fast enough to use mid-conversation

**References:** prototype global search.
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria
<!-- AC:BEGIN -->
- [ ] #1 Searching returns matching clients, bookings and products
- [ ] #2 A record the role cannot view does not appear in results
