## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As the** clinic, **I want** an injectable visit blocked from charting until a consult is linked, **so that** no S4 treatment proceeds without a prescriber's authorisation.

The booking-side half of the compliance gate: a checked-in injectable appointment with no linked consult cannot open the charting screen, and the UI explains exactly what is missing. The consult itself is created in the prescribing epic; this story enforces the precondition at the booking/visit boundary.

**Surface:** web (staff) · API · **Touches:** appointment->consult link, charting precondition.

**Build steps (functional):**
- [ ] Detect whether an injectable visit has a linked, valid consult
- [ ] Block opening charting when it does not, with a clear reason
- [ ] Allow charting once a consult is linked
- [ ] Enforce on the API, not just the UI

**Prototype:** charting entry / consult-gate.
**References:** PRD-02 §6 (consult gate), REQ-BOOK-5, AC2; pairs with PRD-04 consult.
