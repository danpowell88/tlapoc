## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As an** owner, **I want** integrations to be swappable and to keep data within AU/APP-8 bounds, **so that** vendors can change and data never leaks across borders unlawfully.

A ports-and-adapters posture so providers are swappable without touching core modules, and a sub-processor residency control: no integration sends PII to a non-AU sub-processor unless an APP-8 assessment + consent record exists.

**Surface:** API · **Touches:** integration ports/adapters, sub-processor residency gate.

**Build steps (functional):**
- [ ] Integrations behind swappable adapters; core modules unchanged on swap
- [ ] Block PII flow to a non-AU sub-processor without an APP-8 assessment + consent
- [ ] Record integration connections and sync logs

**References:** PRD-10 §6, REQ-INT-3, C21, AC3/AC4, ADR-0012/0016.
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria
