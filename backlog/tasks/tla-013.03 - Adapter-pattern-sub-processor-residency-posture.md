## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As an** owner, **I want** every external provider — auth, payments, calendar, accounting, messaging — swappable and selected by configuration, and all data kept within AU/APP-8 bounds, **so that** we can change vendors without a rewrite and never leak data across borders unlawfully.

Extends the foundational pluggable-provider architecture (TLA-001.13) across all external integrations: each capability sits behind a swappable adapter chosen by per-tenant configuration, so a clinic can run a different identity, payment, calendar or accounting vendor without touching core modules. Plus the sub-processor residency control: no integration sends PII to a non-AU sub-processor unless an APP-8 assessment + consent record exists.

**Surface:** API · **Touches:** provider adapters (auth, payments, calendar, accounting, messaging), per-tenant provider config, sub-processor residency gate.

**Build steps (functional):**
- [ ] All external capabilities (identity/auth, payments, calendar, accounting, messaging, e-prescribing) are behind swappable adapters
- [ ] The provider for each capability is chosen by per-tenant configuration
- [ ] Swapping or adding a provider changes only its adapter, not core modules
- [ ] Block PII flow to a non-AU sub-processor without an APP-8 assessment + consent
- [ ] Record integration connections and sync logs

**References:** PRD-10 §6, REQ-INT-3, C21, AC3/AC4, ADR-0012/0016; realises the foundational provider architecture (TLA-001.13) for all integrations incl. auth.
