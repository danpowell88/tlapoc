## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As an** owner, **I want** visit-based and membership rewards that only ever apply to non-S4 items, **so that** I drive repeat visits without eroding margin or breaching advertising law.

Visit-based rewards (milestones / every-Nth-visit) and membership/frequency perks redeemable on non-S4 items, add-ons or account/gift credit only. Margin-aware rules (value caps, eligible items). A hard invariant: the engine refuses to earn, redeem or discount against any S4 item, and configuring an S4 reward is blocked.

**Surface:** web (staff) · client · API · **Touches:** reward rules/ledger, S4-eligibility block, checkout apply.

**Build steps (functional):**
- [ ] Visit-based rewards and membership perks on non-S4 items only
- [ ] Margin-aware rules: value caps, eligible items
- [ ] Block earn/redeem/discount on any S4 item; block configuring an S4 reward
- [ ] Auto-apply eligible rewards at checkout
- [ ] Surface reward-cost vs retention (owner-only)

**References:** PRD-06 §6, REQ-MEMB-4..7, C9, AC3/AC4, ADR-0014.
