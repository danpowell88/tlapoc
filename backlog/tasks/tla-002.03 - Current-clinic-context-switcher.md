## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As a** staff member who works across locations, **I want** to see and switch which clinic I am working in, **so that** bookings, clients and stock always reflect the right location.

A clear indicator of the active clinic/location and a way to switch it, with all data on screen scoped to the selection. Single-location clinics see it fixed and unobtrusive; multi-location is a later epic but the context primitive belongs here.

**Surface:** web (staff) · **Touches:** location context, data scoping.

**Build steps (functional):**
- [ ] The active clinic/location is always visible
- [ ] Staff with access to more than one location can switch
- [ ] Switching re-scopes the data shown to that location
- [ ] Single-location clinics see it fixed and out of the way

**References:** prototype clinic switch; PRD-01 (Location).
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria
<!-- AC:BEGIN -->
- [ ] #1 The active location is always visible and re-scopes data when switched
- [ ] #2 A single-location user is not burdened by the switcher
