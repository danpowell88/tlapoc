## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As an** RN, **I want** to administer S4 only against a valid script for this client and record exactly what I gave, **so that** the medicine register is complete and provably compliant.

The administration invariant: an administration cannot persist without a valid, unexpired, unconsumed script for the same client, current consent, and a selected in-date stock lot. It records product, units, site, depth and batch-lot/expiry; once saved it is immutable and appears in the audit-ready medicine register, with a lot-to-client link for recall. Saving decrements stock and triggers vial reconciliation.

**Surface:** web (staff) · provider app · API · **Touches:** administration record (immutable), script/consent/lot checks, register, stock decrement.

**Build steps (functional):**
- [ ] Block administration without a valid unconsumed script for the same client + current consent + in-date lot
- [ ] Record product, units, site, depth and batch-lot/expiry
- [ ] Make the saved administration immutable and add it to the register
- [ ] Decrement stock and trigger vial reconciliation
- [ ] Create the lot-to-client link for recall

**References:** PRD-04 §6, REQ-RX-3/MED-4, C8, AC3/AC4; pairs with PRD-05 charting.
