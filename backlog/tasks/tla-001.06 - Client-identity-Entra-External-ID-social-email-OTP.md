## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As a** client, **I want** to create an account with Google/Apple, email+password, or an email/SMS code, **so that** I can access my bookings, intake and records easily.

Wires client-facing authentication (consumer identity) supporting social, email+password and one-time-code sign-in. Client profiles are tenant-scoped. Used by online booking, intake/consent links and (later) the client app.

**Surface:** client web/app · API · **Touches:** Entra External ID, client profile.

**Build steps (functional):**
- [ ] Clients can register and sign in via social (Google/Apple)
- [ ] Clients can register and sign in via email + password
- [ ] Clients can sign in via a one-time email/SMS code
- [ ] A client profile is created and scoped to the right tenant

**References:** PRD-01 §4/§6 (identity), AC3 · spike: Entra External ID <-> Flutter <-> .NET.
