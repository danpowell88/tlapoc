## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As an** engineer, **I want** logging/tracing/metrics and a security baseline in place from day one, **so that** we can operate safely and catch problems early.

The operational and security floor: structured logging, tracing, metrics and alerting; plus encryption in transit and at rest, secure headers, least-privilege access, and dependency/secret scanning in the pipeline. Not a feature — the conditions under which features are safe to run.

**Surface:** infra · API · **Touches:** logging/tracing/metrics, encryption, scanning.

**Build steps (functional):**
- [ ] Structured logs, traces and key metrics are collected with alerting on failures
- [ ] Data is encrypted in transit and at rest
- [ ] Secure HTTP headers and least-privilege access are applied
- [ ] Dependency and secret scanning run in the pipeline and can block a merge

**References:** PRD-01 §6 (security/residency), C10/C21.
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria
<!-- AC:BEGIN -->
- [ ] #1 Errors and key request metrics are observable with alerting
- [ ] #2 Encryption in transit and at rest is enforced
- [ ] #3 A leaked secret or vulnerable dependency is caught by pipeline scanning
<!-- AC:END -->
