## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As a** front desk, **I want** a waitlist that backfills cancellations, **so that** freed slots get filled and revenue is not lost to gaps.

Clients can be added to a waitlist for a service/practitioner/time window; when a matching slot opens (cancel/no-show), the system surfaces or offers it to matching waitlisted clients. Matching respects scope (an injectable offer only to a slot a cleared practitioner can fill).

**Surface:** web (staff) · API · **Touches:** waitlist, slot-open matching.

**Build steps (functional):**
- [ ] Add a client to a waitlist with their service/practitioner/time preferences
- [ ] When a slot frees up, match it to suitable waitlisted clients
- [ ] Offer/assign the slot and track the offer outcome
- [ ] Matching respects scope and resource availability

**References:** PRD-02 §6 (waitlist), REQ-BOOK-4, AC4.
