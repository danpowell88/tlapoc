## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As a** staff member, **I want** the web app to load behind sign-in with the main navigation frame in place, **so that** feature screens have a consistent home to be built into.

The staff web shell: routing, an auth guard that requires a valid session, and the page layout/navigation chrome. The cross-cutting nav, Today dashboard and role context are detailed in the Platform shell epic; this story is the technical shell they slot into.

**Surface:** web (staff) · **Touches:** routing, auth guard, layout.

**Build steps (functional):**
- [ ] App routes are defined with a default authenticated landing route
- [ ] Unauthenticated users are sent to sign-in (auth guard)
- [ ] A persistent layout/nav frame hosts feature screens
- [ ] A placeholder screen renders inside the shell end-to-end

**References:** PRD-01 (Angular/.NET, ADR-0005).
