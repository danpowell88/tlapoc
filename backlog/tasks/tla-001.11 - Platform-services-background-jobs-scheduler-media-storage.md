## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As an** engineer, **I want** a background-job/scheduler mechanism and secure media storage, **so that** later features (reminders, recall, cold-chain checks, before/after photos) have the plumbing they need.

Shared platform services used across many epics: a reliable way to run scheduled and background work (e.g. send a reminder, run a no-show sweep, check a fridge), and secure media storage with time-limited access links for clinical photos and documents.

**Surface:** API · infra · **Touches:** job scheduler, object/media storage, signed access links.

**Build steps (functional):**
- [ ] Background and scheduled jobs can be enqueued and run reliably with retries
- [ ] Media (images/documents) can be stored and retrieved securely
- [ ] Media is served via time-limited, access-controlled links, not public URLs
- [ ] Failures surface in observability

**References:** PRD-01; used by PRD-05 (photos), PRD-07 (reminders), PRD-04 (cold-chain).
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria
<!-- AC:BEGIN -->
- [ ] #1 A scheduled job runs at its due time and retries on failure
- [ ] #2 An uploaded image is retrievable only via a time-limited access link
