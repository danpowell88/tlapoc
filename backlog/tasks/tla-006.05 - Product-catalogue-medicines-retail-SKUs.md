## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As an** owner, **I want** a typed product catalogue with the S4 flag and regulatory class, **so that** scope, rewards eligibility, public naming and prohibited-product blocks all key off one source of truth.

A multi-product, multi-unit catalogue: typed products (toxin/filler/skin) each with their own unit (units vs syringes), plus retail non-S4 SKUs. Capability-gated product admin sets the S4 flag — the single classification that drives rewards eligibility and public booking-page naming. Carries regClass/ARTG/compounded so the system blocks prohibited compounded GLP-1 (banned 1 Oct 2024).

**Surface:** web (staff/admin) · API · **Touches:** product catalogue, S4 flag, regClass/ARTG/compounded, units.

**Build steps (functional):**
- [ ] Typed products (toxin/filler/skin) each with their own unit
- [ ] Capability-gated product admin sets the S4 flag
- [ ] Capture regClass/ARTG/compounded; block prohibited compounded GLP-1
- [ ] Support retail (non-S4) SKUs
- [ ] S4 flag drives rewards eligibility and public naming policy

**References:** PRD-04 rev-2/4 (catalogue), REQ-MED-11/12, ADR-0014/0021; prohibited GLP-1.
