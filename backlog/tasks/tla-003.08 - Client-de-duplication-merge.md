## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As a** front-desk, **I want** to be warned about likely duplicate clients and able to merge them, **so that** one person does not end up with split history across two records.

Duplicate detection at create time (matching name/DOB/contact) and a safe merge that consolidates two records into one, preserving bookings, clinical history and payments. Merges are auditable and, ideally, reversible.

**Surface:** web (staff) · API · **Touches:** duplicate detection, merge transaction.

**Build steps (functional):**
- [ ] Warn when a new client likely duplicates an existing one
- [ ] Merge two clients into one, consolidating their history
- [ ] No bookings, clinical records or payments are lost in a merge
- [ ] The merge is audited

**References:** PRD-02 (client dir merge); PRD-01 client record.
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria
<!-- AC:BEGIN -->
- [ ] #1 Creating a likely-duplicate client surfaces a warning before saving
- [ ] #2 Merging two clients preserves all bookings, clinical history and payments and is audited
