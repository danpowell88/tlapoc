## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As a** client, **I want** to consent to photo use separately and withdraw it any time, **so that** I control whether my before/after images are used and can revoke that later.

A distinct, scoped image-use consent — separate from treatment consent — that the client can grant and withdraw at any time. Withdrawal immediately stops downstream use (e.g. marketing, sharing) and is audited. Whether image use is permitted is surfaced to staff wherever photos are handled.

**Surface:** client web/app · web (staff) · API · **Touches:** image consent (scope, granted/withdrawn), downstream-use enforcement.

**Build steps (functional):**
- [ ] Capture image-use consent separately from treatment consent, with scope
- [ ] Client can withdraw it at any time
- [ ] Withdrawal immediately stops further use and is audited
- [ ] Image-use status is visible wherever staff handle photos

**Prototype:** image-use consent toggle.
**References:** PRD-03 §6 (image consent), REQ-CONS-5, C14, AC4; enforced with PRD-05.
<!-- SECTION:DESCRIPTION:END -->
