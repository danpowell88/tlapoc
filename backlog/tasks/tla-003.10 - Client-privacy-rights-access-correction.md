## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As a** client, **I want** to access my own personal and health information and request a correction, **so that** my records are accurate and my privacy rights are met.

Supports patient access and correction (APP 12/13): a client can view the data held about them and submit a correction request that is tracked to resolution. A collection notice and consent are captured at sign-up.

**Surface:** client web/app · web (staff) · API · **Touches:** access export, correction requests.

**Build steps (functional):**
- [ ] A client can view the data held about them
- [ ] A client can submit a correction request
- [ ] Correction requests are tracked by staff to resolution
- [ ] A collection notice + consent are recorded at sign-up

**References:** PRD-01 §6/§8 (privacy), C21, AC6.
