## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As an** injector, **I want** to drop injection points on a facial diagram or photo, each capturing product, units, depth, technique and batch-lot, **so that** the record is precise and every unit ties back to its vial.

Charting opens with product + batch(lot) selection — restricted to in-date / ARTG / in-custody stock — before mapping. Then tap-to-add and drag injection points on a facial diagram and/or the patient photo; each point captures product, units, depth, technique and batch-lot/expiry. On finalise, charted units deduct from the selected lot and the per-point lot feeds the medicine register and recall lookup. Manual mapping only in v1 (auto-detect deferred to Phase 2).

**Surface:** web (staff) · provider app · API · **Touches:** injection points (coords, product/units/depth/technique/lot), lot selection, stock deduction.

**Build steps (functional):**
- [ ] Select product + in-date/ARTG/in-custody lot before mapping
- [ ] Tap-to-add and drag injection points on a diagram and/or photo
- [ ] Each point captures product, units, depth, technique and batch-lot/expiry
- [ ] On finalise, units deduct from the selected lot
- [ ] Per-point lot links to the medicine register and recall

**Prototype:** clinical-body.png / charting.png — mapping canvas.
**References:** PRD-05 §6 (mapping), REQ-CLIN-2, C8, AC1; no AI in v1 (ADR-0020).
