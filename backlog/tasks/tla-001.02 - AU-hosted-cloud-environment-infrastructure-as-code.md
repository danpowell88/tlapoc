## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As an** engineer, **I want** reproducible cloud environments hosted in Australia, defined as code, **so that** we can stand up/tear down dev, staging and production safely and prove data residency.

All client PII/PHI storage and compute must resolve to Australia East (compliance C21/residency). Environments are defined as code so they are reviewable and repeatable, not hand-built.

**Surface:** infra (cloud + IaC) · **Touches:** environment definitions, networking, secrets store.

**Build steps (functional):**
- [ ] Dev, staging and production environments, all in an Australian region
- [ ] Environment defined as code and applied through the pipeline
- [ ] Secrets/configuration managed centrally, not in source
- [ ] A documented way to verify where data physically resides
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria
<!-- AC:BEGIN -->
- [ ] #1 All PII/PHI storage and compute resolve to an Australian region
- [ ] #2 An environment can be recreated from code with no manual portal clicks
- [ ] #3 Secrets are sourced from a managed store, never from the repo
<!-- AC:END -->
