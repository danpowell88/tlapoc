## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As a** compliance officer, **I want** to view and export the audit trail of who accessed or changed sensitive data, **so that** I can answer an audit or investigate an incident.

The read side of the append-only audit infrastructure: a filterable viewer (by person, client, record type, date) and an export, available to authorised roles only. Viewing the audit log is itself audited.

**Surface:** web (staff) · API · **Touches:** audit query, export.

**Build steps (functional):**
- [ ] Authorised roles can browse audit events, filtered by who/what/when/client
- [ ] The filtered set can be exported
- [ ] Access to the audit log is itself audited

**References:** PRD-01 §6/§8 (audit), C10, AC4.
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria
<!-- AC:BEGIN -->
- [ ] #1 An authorised role can filter and view audit events
- [ ] #2 The audit trail can be exported
- [ ] #3 Viewing/exporting the audit log is itself recorded
