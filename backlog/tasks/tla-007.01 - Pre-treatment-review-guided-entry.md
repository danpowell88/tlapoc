## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As an** injector, **I want** a pre-treatment review screen before I start, **so that** I confirm safety, the last treatment and that consult/consent are in place at a glance.

The guided entry to charting: surfaces the safety picture (contraindications, BDD flag, allergies), the last treatment, and confirmation that the consult, prescription and current consent are present. It is the human checkpoint that complements the server-side gate before mapping begins.

**Surface:** web (staff) · provider app · API · **Touches:** pre-treatment summary across intake/consult/consent.

**Build steps (functional):**
- [ ] Summarise safety flags (contraindications, BDD, allergies)
- [ ] Show last treatment and any plan/protocol in progress
- [ ] Confirm consult, prescription and current consent are present
- [ ] Provide the entry point to start charting only when clear

**Prototype:** clinical-safety.png / charting entry.
**References:** PRD-05 rev-2 (guided flow), REQ-CLIN-9, ADR-0024.
<!-- SECTION:DESCRIPTION:END -->
