## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As a** prescriber, **I want** to write an individual prescription tied to one client and one consult, **so that** every S4 administration traces to a lawful, patient-specific script.

An individual Prescription per client per consult. The system rejects any attempt to apply one script to multiple clients or to create a standing-order/batch/async script. An off-label flag requires consent that covers off-label use. A prescription cannot be saved without a linked synchronous consult.

**Surface:** web (staff/prescriber) · API · **Touches:** prescription, consult linkage, off-label flag, consent check.

**Build steps (functional):**
- [ ] Write a prescription bound to one client and one consult
- [ ] Reject batch/standing-order/async prescriptions
- [ ] Require a linked synchronous consult to save
- [ ] Off-label flag requires consent covering off-label use
- [ ] Output a structured script + PDF (no ETP e-prescribing in v1)

**References:** PRD-04 §6 (prescribing), REQ-RX-2/5, C2, AC1/AC2/AC8.
