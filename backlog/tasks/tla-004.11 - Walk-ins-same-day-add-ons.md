## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As a** front desk, **I want** to take a walk-in or add a service to an in-progress visit, **so that** I can serve someone who arrives without a booking — without bypassing compliance.

Create a same-day appointment for a walk-in, or add a service to an existing visit, using the live availability engine. Gate-respecting: an injectable walk-in still requires intake/consent and a consult before treatment, exactly like a booked visit. Conflicts on resources are flagged.

**Surface:** web (staff) · API · **Touches:** same-day appointment create, add-on to visit, gates.

**Build steps (functional):**
- [ ] Create a walk-in appointment for the soonest suitable slot
- [ ] Add a service to an in-progress visit
- [ ] Injectable walk-ins still require intake/consent + consult (no bypass)
- [ ] Resource conflicts are flagged

**References:** PRD-02 rev-4 (walk-ins & add-ons), REQ-BOOK-7.
