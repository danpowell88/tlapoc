## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As a** client, **I want** to book online by choosing service, practitioner and time, **so that** I can secure an appointment without calling the clinic.

The public booking surface (web, later the client app). Same scope-aware availability engine as the desk: injectable services offer only RN/NP and present compliant generic service names. Captures date of birth (flagging under-18 for downstream cooling-off) and kicks off intake/consent. Produces the same Appointment entity as desk and walk-in (source=online).

**Surface:** public web · API · **Touches:** appointment create (source=online), availability engine, DOB capture.

**Build steps (functional):**
- [ ] Public flow: service -> practitioner -> slot -> details -> confirm
- [ ] Availability is scope-aware (injectables -> RN/NP only)
- [ ] Public service names follow advertising rules (generic for S4)
- [ ] Capture DOB and flag under-18; trigger intake/consent
- [ ] Confirmation shows the cancellation policy

**Prototype:** public-booking.png / booking-widget.
**References:** PRD-02 §6 (online booking), REQ-BOOK-2, C4, C9, C6 feed.
