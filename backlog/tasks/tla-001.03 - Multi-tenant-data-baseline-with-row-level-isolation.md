## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As an** engineer, **I want** every record scoped to a tenant with isolation enforced at the data layer, **so that** one clinic can never see another clinic's data even if application code has a bug.

Every row carries a tenant identifier; isolation is enforced below the application (row-level security) and the API sets tenant context per request. This is the safety net the whole multi-tenant product relies on.

**Surface:** API · data · **Touches:** data model (tenant scoping), request tenant-context.

**Build steps (functional):**
- [ ] Every persisted record is associated with exactly one tenant
- [ ] Isolation is enforced at the data layer, not only in application code
- [ ] The API resolves and applies the caller's tenant on every request
- [ ] A query for another tenant's record returns not-found, never data

**References:** PRD-01 §6 (tenancy), AC2.
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria
<!-- AC:BEGIN -->
- [ ] #1 A user in Tenant A querying any record receives only Tenant-A rows
- [ ] #2 Requesting a record by an id belonging to another tenant returns not-found
- [ ] #3 Isolation holds even when the application-level filter is omitted (enforced at data layer)
<!-- AC:END -->
