## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As an** injector, **I want** to finalise the note into a locked record and run close-out, **so that** the clinical record is tamper-evident and aftercare/recall are set up before checkout.

Finalisation locks the chart entry (immutable); any later change is an appended, audited amendment that preserves the original. Finalisation happens server-side and links the administration to the medicine register. Close-out sets up aftercare, recall (next-treatment-due), and a 2-day wellbeing call, handing off to checkout.

**Surface:** web (staff) · provider app · API · **Touches:** finalise/lock, amendments, close-out (aftercare/recall/wellbeing).

**Build steps (functional):**
- [ ] Finalise locks the note; it cannot be edited afterwards
- [ ] A later change is an appended, audited amendment preserving the original
- [ ] Finalisation occurs server-side and links to the medicine register
- [ ] Close-out sets aftercare, a recall for next-treatment-due, and a 2-day wellbeing call
- [ ] Hand off to checkout

**Prototype:** charting finalise / close-out.
**References:** PRD-05 §6 (immutability, finalise), REQ-CLIN-4/9, C8, AC3, ADR-0010/0024.
