## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As an** owner, **I want** the system to know in real time whether a practitioner may administer an injectable, **so that** an unqualified, lapsed or uninsured practitioner can never be booked or charted for an S4 treatment.

A single derived gate, canInject, computed from a practitioner's credential, CPD and cosmetic-cover records: a practitioner with lapsed registration, missing experience flag, or insurance that excludes cosmetic is flagged not-bookable for Schedule-4 treatments. Booking availability, the booking wizard and charting all consult this gate.

**Surface:** API · web (staff) · **Touches:** canInject derivation, credential records, booking/charting checks.

**Build steps (functional):**
- [ ] Derive a single canInject result per practitioner from their credential/CPD/cover records
- [ ] A practitioner with lapsed registration or no cosmetic cover is not bookable for S4
- [ ] Booking availability and the booking wizard respect the gate
- [ ] Charting/administration is blocked when the gate is false, with the reason

**References:** PRD-01 rev-4, REQ-TEN-6..9, C4, ADR-0028/0029.
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria

- [ ] #2 canInject reflects credential, CPD and cosmetic-cover status
