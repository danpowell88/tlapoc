## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As a** staff member, **I want** a searchable, filterable client directory, **so that** I can find anyone fast and keep the list clean.

The client list: search by name/contact, filter/segment (e.g. new, VIP, overdue recall), and safe soft-delete with audit (a removed client is recoverable, never hard-deleted). Built on the core client record; duplicate merge lives in the identity epic.

**Surface:** web (staff) · API · **Touches:** client query/search, segments, soft-delete.

**Build steps (functional):**
- [ ] Search clients by name and contact details
- [ ] Filter/segment the list (e.g. new, VIP, recall due)
- [ ] Soft-delete a client with audit; deleted clients are recoverable
- [ ] Open a client to their 360 profile

**Prototype:** clients.png — directory with segments.
**References:** PRD-02 §6 (client/CRM), REQ-CLI-1/2.
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria
<!-- AC:BEGIN -->

- [ ] #2 A soft-deleted client is removed from the default list, audited, and recoverable
