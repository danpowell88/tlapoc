## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As a** staff member, **I want** to sign in through whatever identity provider my clinic uses (Microsoft 365/Entra to start), **so that** I get a secure, scoped session — and the clinic isn't locked to one vendor.

Implements the **identity/auth port** (from the pluggable provider architecture) for staff, with an **Entra ID SSO + MFA adapter shipped first**. Another provider (e.g. Google Workspace, Okta/Auth0, Cognito) is added later as a new adapter with no change to app code. The session is scoped to the staff member's tenant; role/scope enforcement layers on in the identity epic. An owner can invite staff who sign in with the configured provider.

**Surface:** web (staff) · API · **Touches:** identity port, Entra adapter (first), session/tenant context.

**Build steps (functional):**
- [ ] Staff sign-in goes through the identity/auth port, not a hard-coded vendor
- [ ] Ship an Entra ID SSO adapter first, with MFA enforced
- [ ] The resulting session is scoped to the staff member's tenant
- [ ] An owner can invite a staff member who signs in via the configured provider
- [ ] A second staff identity provider can be added as a new adapter without app changes

**References:** PRD-01 §4/§6 (identity), AC3, ADR-0004; sits behind the provider port (TLA-001.13).
