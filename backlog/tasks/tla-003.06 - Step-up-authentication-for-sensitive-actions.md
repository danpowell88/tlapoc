## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As an** owner, **I want** an extra identity check before the most sensitive actions, **so that** prescribing, S4 medicine movements and record exports cannot happen from an unattended session.

Re-authentication / step-up before high-risk actions (e.g. authorising a prescription, recording an S4 stock movement, exporting clinical/PII data). The set of gated actions is configurable.

**Surface:** web (staff) · API · **Touches:** step-up challenge, sensitive-action list.

**Build steps (functional):**
- [ ] Define which actions require step-up
- [ ] Prompt for re-authentication before a gated action proceeds
- [ ] A passed/failed step-up is audited
- [ ] Step-up applies on the API, not just the UI

**References:** PRD-01 §11 (MFA for sensitive actions); used by PRD-04.
<!-- SECTION:DESCRIPTION:END -->
