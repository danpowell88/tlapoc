## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As a** front desk, **I want** to build an itemised bill for a visit and check the client out, **so that** treatment, units and any retail are charged correctly and tied to the record.

The checkout that closes a visit: assemble line items (service, charted units/products, retail) with per-line GST coding, tie the invoice to the visit and client, and proceed to tender. Each line carries the catalogue schedule flag so S4 lines are identifiable and never discountable (the rewards engine, later, enforces this). The client's own bill is operational and visible to the desk; clinic-wide revenue figures remain owner-only.

**Surface:** web (staff) · API · **Touches:** invoice/lines, per-line GST, schedule flag, visit/client link.

**Build steps (functional):**
- [ ] Assemble line items from the visit (service + charted units/products) and retail
- [ ] Per-line GST coding (services and retail both taxable)
- [ ] Tie the invoice to the visit and client
- [ ] Carry the S4/non-S4 schedule flag per line (S4 never discountable)
- [ ] Proceed to tender

**Prototype:** checkout.png.
**References:** PRD-06 §6 (POS), REQ-PAY-2, C9; per-line GST (ADR-0027).
