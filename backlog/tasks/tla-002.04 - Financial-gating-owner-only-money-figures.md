## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As an** owner, **I want** revenue, pricing margin and recurring-revenue figures visible only to me (and roles I grant), **so that** front-desk and clinical staff can do their jobs without seeing the clinic's money.

A cross-cutting capability that gates monetary figures. Revenue, MRR and pricing/margin are shown only to roles holding the financial capability; non-owner roles such as Reception may see that a client has a membership, but never the money. This is a hard product rule enforced server-side, not just hidden in the UI.

**Surface:** web (staff) · API · **Touches:** capability check, every money figure.

**Build steps (functional):**
- [ ] A single financial capability gate that any money figure is checked against
- [ ] Revenue, MRR, pricing and margin are hidden from roles without it
- [ ] Non-financial roles can still see non-money facts (e.g. that a membership exists)
- [ ] Enforced on the API, not only hidden in the UI

**References:** CLAUDE.md financial-gating rule; PRD-08 (owner-only reporting).
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria
<!-- AC:BEGIN -->

- [ ] #2 Money figures are withheld by the API for roles without the financial capability, not just by the UI
