## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As an** owner, **I want** to define the services we offer with their schedule class, duration and eligible roles, **so that** bookings are priced, timed, scope-gated and advertised correctly.

The catalogue every booking surface draws on: each service/treatment type carries a schedule class (S4 prescription-only vs non-S4), real duration and buffers, eligible practitioner roles, and a public-facing name that follows advertising rules (generic names for S4). The schedule class is what drives downstream scope gating and the consult requirement.

**Surface:** web (staff) · API · **Touches:** services/treatment types, schedule class, eligible roles, public naming.

**Build steps (functional):**
- [ ] Define services with duration, buffers and price
- [ ] Mark each service S4 or non-S4 (drives scope + consult gating)
- [ ] Set eligible roles per service
- [ ] Set a compliant public-facing name (generic for S4 per advertising rules)

**Prototype:** settings-booking.png / service cards.
**References:** PRD-02 §6 (online booking, naming), C4, C9 (advertising).
<!-- SECTION:DESCRIPTION:END -->
