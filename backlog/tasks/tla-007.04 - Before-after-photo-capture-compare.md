## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As an** injector, **I want** to take standardised before/after photos room-side and compare them across visits, **so that** outcomes are documented and reviewable.

Provider-app camera or upload with standardised framing/ghosting guidance, side-by-side comparison across visits, and annotation. Photos are gated by current image-use consent, stored centrally via time-limited signed links, and never persisted on personal devices beyond a transient sync cache.

**Surface:** provider app · web (staff) · API · **Touches:** photos (framing meta, consent link), central media storage, compare view.

**Build steps (functional):**
- [ ] Capture or upload photos with standardised framing/ghosting guidance
- [ ] Require current image-use consent before capture/use
- [ ] Store centrally via signed links; never persist on the device beyond sync cache
- [ ] Compare before/after side-by-side across visits with annotation

**Prototype:** clinical-imaging.png — before/after compare.
**References:** PRD-05 §6 (photos), REQ-CLIN-3, C14, AC2/AC6, ADR-0009; uses media storage.
