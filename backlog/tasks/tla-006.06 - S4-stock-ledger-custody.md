## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As an** owner/prescriber, **I want** a stock ledger with exclusive custody and secure storage for S4 medicines, **so that** every vial is accounted for and held lawfully.

The Mode-A on-site stock ledger: receipt, custodian, storage location, administration decrement, wastage, disposal and expiry alerts. Custody is limited to NP/prescriber roles; the custodian must physically work at the clinic and hold exclusive custody, recorded with an exclusive-custody attestation and a designated medicine-store contact. Stock is bound to a secure, access-logged location (locked cabinet/room; the S4 cabinet monitor feeds access/tamper events).

**Surface:** web (staff/prescriber) · API · hardware feed · **Touches:** stock ledger, custody attestation, secure-location access log.

**Build steps (functional):**
- [ ] Ledger movements: receipt, decrement, wastage, disposal, expiry alerts
- [ ] Custody limited to NP/prescriber with an exclusive-custody attestation + store contact
- [ ] Bind stock to a secure location with logged access
- [ ] Ingest S4 cabinet access/tamper events
- [ ] Expiry alerts ahead of lot expiry

**References:** PRD-04 §6 (Mode A), REQ-MED-2, C7/C15, AC5; docs/hardware/esp32-cabinet-sensor.md.
