## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As the** clinic, **I want** treatment blocked until all required intake and consent are complete and current, **so that** compliance is guaranteed by the system, not by staff memory.

The keystone gate: treatment/charting cannot start unless the required intake, BDD screen and current version-matched consent (and, for injectables, a consult) are complete. The block states exactly what is missing. Enforced server-side so it cannot be clicked past.

**Surface:** web (staff) · API · **Touches:** gate evaluation across intake/consent/consult, charting precondition.

**Build steps (functional):**
- [ ] Evaluate completeness of intake, BDD and current consent for a visit
- [ ] Block treatment/charting until all required items are complete and current
- [ ] State precisely what is missing
- [ ] Enforce on the API, not just the UI
- [ ] Combine with the consult requirement for injectables

**References:** PRD-03 §6 (gating), ADR-0008, AC1; pairs with booking consult gate + PRD-04.
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria
<!-- AC:BEGIN -->
- [ ] #1 Treatment is blocked until current, version-matched consent and required intake/BDD are complete
- [ ] #2 The block names exactly what is missing and cannot be bypassed via the API
