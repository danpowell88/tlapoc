## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As a** front desk, **I want** a guided wizard to create an appointment in a few steps, **so that** I can book clients quickly at the desk or over the phone.

The desk/phone counterpart to online self-booking and the path most bookings flow through: service then practitioner then time slot then client then confirm. It uses the same availability engine, so for an injectable only cleared RN/NP appear (plus a 'next available' shortcut). At the client step the desk attaches a returning client or creates one inline; under-18 is flagged; an option sends intake + consent now with 'treatment can't start until complete'. Confirm shows the cancellation policy and schedules the reminder.

**Surface:** web (staff) · API · **Touches:** appointment create (source=desk), client attach/create, intake/consent send trigger.

**Build steps (functional):**
- [ ] Wizard steps: service -> practitioner -> time slot -> client -> confirm
- [ ] Injectable services offer only cleared RN/NP, with a 'next available' shortcut
- [ ] Attach an existing client or create one inline; flag under-18
- [ ] Option to send intake + consent links on booking
- [ ] Confirm shows cancellation policy and schedules a reminder; appointment is source=desk

**Prototype:** booking-wizard.png — 5-step stepper.
**References:** PRD-02 §4/§6, REQ-BOOK-7, C4; intake/consent send -> PRD-03.
