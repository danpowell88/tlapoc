## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As a** staff member, **I want** to be notified in-app of things that need me, **so that** I do not miss a waiting client, a failed payment, a cold-chain alert or a new follow-up.

The in-app notification surface: a list/badge of role-relevant events raised by other features (client checked in, intake completed, payment failed, fridge breach, concern reported). Delivery channels beyond in-app (SMS/email) live in the comms epic.

**Surface:** web (staff) · API · **Touches:** notification feed, event sources.

**Build steps (functional):**
- [ ] A notification feed/badge for the signed-in user
- [ ] Other features can raise a notification to a role or user
- [ ] Notifications deep-link to the relevant record
- [ ] Read/unread state is tracked

**References:** prototype notifications; feeds from PRD-04/05/06/11.
<!-- SECTION:DESCRIPTION:END -->
