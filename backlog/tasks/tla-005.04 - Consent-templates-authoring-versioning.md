## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As an** owner/admin, **I want** to author consent templates with the mandated content and version them, **so that** consents stay compliant and previously signed consents remain bound to what was signed.

The admin side of consent: author/edit consent templates, enforce that mandated content blocks are present before a template can be published, and version on every change. Changing a template creates a new version; existing signatures stay bound to their original version.

**Surface:** web (staff/admin) · API · **Touches:** consent templates, mandated-content check, versioning.

**Build steps (functional):**
- [ ] Author and edit consent templates
- [ ] Block publishing a template missing mandated content blocks
- [ ] Editing a published template creates a new version
- [ ] Previously signed consents remain bound to their signed version

**References:** PRD-03 §6 (consent versioning), REQ-CONS-1, C5, AC6.
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria
<!-- AC:BEGIN -->
- [ ] #1 A template cannot be published while mandated content is missing
- [ ] #2 Editing a published template creates a new version and does not alter past signatures
