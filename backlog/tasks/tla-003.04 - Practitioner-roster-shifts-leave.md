## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As a** manager, **I want** to set each practitioner's working shifts and time off, **so that** the calendar only offers appointments when someone is actually available.

The roster that booking availability is built on: per-practitioner shifts by day/location and time-off/leave. Availability everywhere is the intersection of roster, the canInject scope gate, and a free room/chair/device — never raw roster alone.

**Surface:** web (staff) · API · **Touches:** roster shifts, leave, availability source.

**Build steps (functional):**
- [ ] Set recurring and one-off shifts per practitioner and location
- [ ] Record time off / leave that blocks availability
- [ ] Expose roster as the availability source booking consumes
- [ ] Unrostered periods are not bookable

**References:** PRD-01 rev-4 (roster), ADR-0028/0029; consumed by PRD-02 calendar.
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria
<!-- AC:BEGIN -->
- [ ] #1 A practitioner is only offered for booking inside a rostered shift
- [ ] #2 Time off removes that practitioner's availability
