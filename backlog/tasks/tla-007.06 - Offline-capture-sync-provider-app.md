## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
**As an** injector working in the room, **I want** my notes and photos to queue locally and sync when back online, **so that** nothing is lost if connectivity drops mid-visit.

Offline-tolerant charting on the provider app: notes and photos queue locally (encrypted) when offline and sync without loss when connectivity returns. Finalisation always occurs server-side once synced. Conflict/integrity handling proven by the offline-sync spike.

**Surface:** provider app · API · **Touches:** local queue (encrypted), sync, server-side finalise.

**Build steps (functional):**
- [ ] Notes and photos queue locally when offline (encrypted)
- [ ] Queued items sync with no loss when back online
- [ ] Finalisation occurs server-side after sync
- [ ] Sync conflicts/integrity are handled per the spike findings

**References:** PRD-05 §6, ADR-0015, AC4; depends on offline-sync spike.
